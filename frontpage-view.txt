WITH MISSING DATA WARNING:
--------------------------
Models missing data (e.g., grok-test-bias or claude-test-bias) are visually indicated with:

- Dashed, semi-transparent lines (~60% opacity)
- Model badges faded (~50% opacity) with a warning icon (⚠️)
- Tooltips explain: "Missing data for {model} so user does not think it scores that poorly"
- A clear alert box is displayed above or beside the chart listing all missing data warnings

WITH DATA INCONSISTENCY ALERTS:
-------------------------------
Models with large score discrepancies between different charts (e.g., timeline vs. radar chart) show:

- Red warning points on the graph
- Model label appended with "(⚠️ Inconsistent)"
- Tooltip warnings on hover describing inconsistency details
- An alert box or modal dialog details the inconsistency, including exact score differences and explanations
- Users can click "Show details" to see a full breakdown of inconsistencies and missing data

   Critical alerts (red) > Warnings (amber) > Info (blue)
   
3. PROGRESSIVE DISCLOSURE:
   Summary alerts → Click for details → Full validation report
   
4. CONSISTENT MESSAGING:
   Same validation message across all chart types and tooltips

   Real-time validation, no waiting for page reloads
   
6. ACCESSIBILITY:
   Color coding + text labels + icons for all states

================================================================================
🔧 TECHNICAL VALIDATION BEHAVIOR
================================================================================

DATA COMPLETENESS CHECKS:
-------------------------
✓ Missing bias scores for any model
✓ Incomplete time series (< 7 days)
✓ Empty category data in radar charts
✓ Missing drift data for bias categories
✓ Invalid score ranges (< 0 or > 100)

CONSISTENCY VALIDATIONS:
-----------------------
✓ Timeline vs Radar score differences > 30 points
✓ Models present in one chart but missing in another
✓ Suspiciously uniform data patterns
✓ Score validation across multiple data sources

ALERT GENERATION LOGIC:
----------------------
1. Load chart data from API
2. Run all validation functions
3. Generate appropriate alert objects
4. Apply visual styling to charts
5. Display user-facing warnings
6. Log technical details to console

This comprehensive validation system ensures users always understand data quality 
and never misinterpret missing data as poor model performance.
